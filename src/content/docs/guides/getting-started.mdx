Get the Among Us experience in your Minecraft server — follow the steps below to get up and running quickly.

## Prerequisites

- A running **Paper** (or compatible) Minecraft server
- Server operator / file system access

---

## Installation

<Steps>

    1. **Download the plugin.**

        Choose your preferred method:

        <Tabs>
            <TabItem label="Download (Recommended)">
                1. Open the [Releases](https://github.com/Fantamomo/among-us-in-minecraft/releases/) page on GitHub.
                2. Expand **Assets** on the latest release.
                3. There are now 3 .jar files:
                    <Tabs>
                        <TabItem label="lite (Recommended)">
                            The lite version is the recommended option for most users.

                            On first startup, it will download all required dependencies automatically.
                            This means the initial startup may take approximately **1–3 minutes**.

                            An internet connection is required only for the first launch.
                        </TabItem>
                        <TabItem label="standalone">
                            All dependencies are bundled directly into the plugin.

                            This results in a larger file size, but no additional downloads are required at startup.
                        </TabItem>
                        <TabItem label="thin">
                            :::danger[Does not work]
                            This version **does not work**. It is only intended for special cases and should not be used in normal setups.

                            The plugin will log a warning and throw an exception on startup.
                            :::
                        </TabItem>

                    </Tabs>
            </TabItem>
            <TabItem label="Build from Source">
                Clone the repository and build the plugin using Gradle:

                ```bash
                git clone https://github.com/Fantamomo/among-us-in-minecraft
                cd among-us-in-minecraft
                ```

                This plugin depends on packages hosted on **GitHub Packages**, which requires authentication — even for read access.

                Create or edit your `~/gradle.properties` file and add your GitHub credentials:

                ```properties
                gpr.user=YOUR_GITHUB_USERNAME
                gpr.key=YOUR_GITHUB_TOKEN
                ```

                :::tip[Creating a token]
                <Steps>
                    1. Open [GitHub Tokens](https://github.com/settings/tokens)
                    2. Click **Generate new token**
                    3. Select `read:packages`.
                    4. Click **Generate token**.
                    5. Place it in `~/gradle.properties`
                </Steps>
                :::

                Then build the plugin:

                ```bash
                ./gradlew shadowJar
                ```

                The compiled `.jar` will be located in `build/libs/`.

                :::note
                The authentication requirement for GitHub Packages is a known inconvenience. We hope to migrate the dependencies to a public registry in the future to simplify this process.
                :::
            </TabItem>
        </Tabs>

    2. **Add the plugin to your server.**

        Move the downloaded `.jar` file into your server's `plugins/` directory:

        <FileTree>
        - plugins/
          - **AmongUs-x.x-SNAPSHOT-x.jar** the downloaded .jar file
          - ...
        - ...
        </FileTree>

    3. **Start your server.**

        If your server is already running, stop it first, then start it again.

        :::caution
        This plugin does **not** support `/reload`. Always perform a full restart — both on first install and after every update.
        :::

    4. **Verify the installation.**

        Once the server is running, check your console for a line like:

        ```log
        [AmongUs] Enabling AmongUs vX.X-SNAPSHOT
        ```

        If you see it, you're all set! Next, adjust your [config.yml](/configuration/config-yml) to your liking or
        jump straight into [creating your first area](../create-an-area) or
        download the [pre-built-area](/demo/map).

</Steps>
