This plugin provides a `config.yml` file located at:

```
/plugins/AmongUs/config.yml
```

Below is the **default configuration**:

```yaml
msg-command-blocker:
  disabled: false
  legacy: false
  commands:
    - msg
    - tell
    - w
    - minecraft:tell
    - minecraft:msg
    - minecraft:w

morph-blender:
  enabled: false
  blender: virus
  visibility: auto

roles:
  disabled:
  #  - phantom

modifications:
  disabled:
  #  - small

game-creation:
  everyone-can-create: false
  max-games: 10
  ignore-admins: true

animate-scoreboard: true
```

## Msg-Command-Blocker

Private messaging can break gameplay by allowing hidden information to be shared. This module ensures fair play by intercepting commands before they execute.

### How It Works

Private message commands (e.g., `/msg`, `/tell`, `/w`) are **blocked** if:

* The **sender** is in a game
* At least one **target player** is in a game

Commands are **allowed** if:

* Sender and target are the same player
* Neither player is in a game
* The module is disabled

### Configuration Options

#### disabled

```yaml
disabled: false
```

* `false` → Blocking is active
* `true` → No commands are blocked

#### legacy

```yaml
legacy: false
```

* `false` → Automatically intercepts private message commands (recommended)
* `true` → Only commands listed under `commands:` are blocked

:::danger
Legacy mode can only check the sender, not the target.
This means players outside a game can still message players inside a game.
:::

:::caution
Enable legacy mode if:
* Another plugin overrides `/msg`
* Custom private message commands are registered
* You receive console warnings related to command interception
:::

#### commands

Only used when `legacy: true`.

```yaml
commands:
  - msg
  - tell
  - w
```

Add additional aliases if your server uses them.

## MorphBlender

:::info
MorphBlender enables animated skin transitions when a player changes form.
:::

A valid **MineSkin API key** must be configured in `secrets.properties`.

If disabled, players switch skins instantly without animation.

### Configuration Options

#### enabled

```yaml
enabled: false
```

* `true` → Animated morph transitions
* `false` → Instant skin switch

#### blender

```yaml
blender: virus
```

Available options:

* `virus` – Viral spreading animation
* `fade` – Smooth fade transition
* `checker` – Checkerboard blending

#### visibility

```yaml
visibility: auto
```

Available options:

* `public` – Visible to everyone (not recommended)
* `unlisted` – Accessible via link only
* `private` – Only visible to your MineSkin account
* `auto` – Automatically selects best visibility based on your plan (recommended)

## Roles

Defines roles that are disabled and hidden from gameplay.

```yaml
roles:
  disabled:
    - phantom
```

* Roles listed under `disabled:` cannot be selected
* Disabled roles are completely hidden in-game

```yaml
modifications:
```

## Game Creation

Controls who can create games and how many games can exist simultaneously.

```yaml
game-creation:
  everyone-can-create: false
  max-games: 10
  ignore-admins: true
```

### Configuration Options

#### everyone-can-create

```yaml
everyone-can-create: false
```

* `true` → All players can create games
* `false` → Only players with permission `amongus.admin.game.create` can create games

#### max-games

```yaml
max-games: 10
```

Defines the maximum number of games that can exist at the same time.

If the limit is reached, no additional games can be created.

#### ignore-admins

```yaml
ignore-admins: true
```

* `true` → Players with `amongus.admin.game.create` can bypass the `max-games` limit
* `false` → Even admins are restricted by `max-games`

---

## animate-scoreboard

```yaml
animate-scoreboard: true
```

* `true` → Scoreboard updates dynamically with animation
* `false` → Static scoreboard

## Recommended Setup

For most servers:

```yaml
msg-command-blocker:
  disabled: false
  legacy: false

morph-blender:
  enabled: true
  blender: virus
  visibility: auto

roles:
  disabled: []

modifications:
  disabled: []

game-creation:
  everyone-can-create: false
  max-games: 10
  ignore-admins: true

animate-scoreboard: true
```
